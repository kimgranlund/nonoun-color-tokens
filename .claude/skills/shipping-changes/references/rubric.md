## Rubric — a shipped change

Scores a ship of ultimate-tokens (one PR landed + local synced). `[gate]` = mechanically checkable
(grep / a `gh`/`git` command); `[review]` = judgment with cited evidence. Score each 1–5.

| # | Dimension | Type | What it checks | 1 (fail) → 3 (adequate) → 5 (excellent) |
|---|---|---|---|---|
| H1 | Branch hygiene | [gate] | Work was on a feature branch cut from fresh `origin/main`, not a direct `main` commit | 1: committed on `main` or off a stale base · 3: branched off origin/main · 5: branched off a freshly `fetch`ed origin/main |
| H2 | Gates run | [gate] | `npm test` green pre-push; `npm run build` run iff the build chain (TS/vite/scripts/fonts) was touched | 1: pushed without `npm test`, or skipped build after a build-chain edit · 3: test green, build judgment ok · 5: both correct + green, build deliberately skipped for a pure-logic change |
| H3 | Commit guards | [gate] | `git status --short` showed no `.claude/docs/other/` and no `node_modules`; no hand-edited generated artifacts | 1: `.claude/docs/other/` or `node_modules` committed, or a hand-edited `ui.html`/`*-assets.js` · 3: clean · 5: clean + artifacts regenerated by the gate, count gates moved together |
| H4 | Trailers + PR title | [gate] | Commit ends with the `Co-Authored-By: Claude Opus 4.8 (1M context)` trailer; PR body ends with the 🤖 line; PR **title** is a conventional-commit changelog subject | 1: trailer/🤖 missing or title is "wip"/"fixes" · 3: all present · 5: present + title reads as the real changelog line |
| H5 | CI watched, all three legs | [review] | CI passed build · test · **smoke**; for a UI change, the `smoke-screenshots` artifact was looked at | 1: merged red or without watching CI · 3: all three green · 5: green + screenshot inspected (and Safari reasoned-about if WebKit-relevant) |
| H6 | Merge + sync | [gate] | Squash-merged; merge confirmed via `gh pr view --json state,mergedAt`; local `main` fast-forwarded; branch deleted locally (`-D`) and on remote | 1: not merged, or local main not synced, or branch left dangling · 3: merged + synced · 5: merged + ff-only sync + both branches deleted + gh-quirk handled correctly |
| H7 | Concurrency isolation | [review] | If the tree was churning under a fan-out, work was isolated in a worktree (symlinked `node_modules`, only-your-files), not stashed/committed in the shared tree; collision files re-applied from clean main | 1: stashed/committed in the churning tree, swept foreign hunks, or hand-merged `ui.html` · 3: isolated correctly · 5: isolated + collision files (`headless-boot.mjs` count, `ui.html` bundle) handled + clean worktree teardown · N/A if single-agent quiet tree |

**Gate to ship:** H1, H2, H3, H4, H6 must each score ≥ 3 (and H7 ≥ 3 whenever a fan-out was in play). A
ship that committed on `main` (H1), skipped a required gate (H2), leaked `.claude/docs/other`/`node_modules` (H3),
dropped the trailer (H4), or left local `main` unsynced / a branch dangling (H6) is not landed regardless
of how good the diff is.

**Top failure to look for first:** treating **green Chrome smoke as Safari proof** (H5) — the user previews
in Safari and WebKit is stricter; a passing CI run is not cross-browser verification. Second: a `git branch
-D` done **before** confirming the squash actually landed on `main` (H6) — the work-loss trap.
