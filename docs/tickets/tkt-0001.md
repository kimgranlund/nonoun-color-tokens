# TKT-0001 — Figma text-style names are asymmetric: the core weight carries no segment, siblings are title-case

## Summary
The Figma export names a voice/step's CORE weight style bare (`Display/xs`) while its sibling weight
variants carry a title-case name segment (`Display/xs/Extra-Bold`, `Display/xs/Bold`) — inconsistent
and implicit about which entry is the default. Make every entry, including the core, carry an explicit,
lowercase-kebab weight segment.

## Acceptance
- A voice/step WITH configured weight siblings emits `{Voice}/{step}/{weight-slug}` for every entry,
  including the core — e.g. core weight 900 (Black) + siblings Extra-bold/Bold → `Display/xs/black`,
  `Display/xs/extra-bold`, `Display/xs/bold`.
- The weight-slug segment is lowercase-kebab for every entry (core AND siblings) — siblings reuse the
  already-existing `wv.slug` field; the core's slug is newly derived by snapping its weight to the
  nearest `WEIGHT_LADDER` stop (the same snap `siblingWeightDefaults` uses).
- A voice/step WITHOUT any configured siblings keeps the current bare `{Voice}/{step}` name — nothing to
  disambiguate, no needless segment added.
- No change to the CSS/DTCG weight-token KEYS (`--type-{voice}-weight-{slug}`, DTCG `weights.{voice}.{Name}`)
  or the human-readable `styleName`/weight-style-name field (still title case, used for non-variable font
  matching) — this ticket is scoped to the Figma-facing style PATH only.

## Links
- Builds on #263 (adjacent weight siblings) and #231-236 (the sibling-weight/text-style feature).

## Repro
1. Give a voice a core weight near a ladder extreme (e.g. Display at weight ~900) with siblings enabled
   (either via the editor's "Suggest" button or a Color Categories preset that ships them by default,
   per #263).
2. Export to Figma / inspect the Styles panel.
3. Observed: a flat `Display` folder with `xs · sm · md · lg · xl` (the core, unlabeled), then a nested
   `Display/xs` folder containing `Extra-bold` and `Bold` (title case) — see attached screenshot.

## Expected vs actual
- **Expected**: `Display/xs/black`, `Display/xs/extra-bold`, `Display/xs/bold` — every entry explicit
  and symmetrically formatted.
- **Actual**: `Display/xs` (bare, no weight segment) + `Display/xs/Extra-Bold` + `Display/xs/Bold`
  (title case, inconsistent casing with the rest of this app's token-naming convention).

## Classification
Structural / interface — the exported artifact's naming CONTRACT (`figma/binder/style-plan.mjs`'s
`stylePlans()`), not a data or rendering defect: every underlying value (size, weight, line-height) is
already correct; only the constructed `name` string is asymmetric. Component: the text-style planner's
`texts` array construction (core-style + sibling-style name segments).

## Severity
minor — a real usability defect in a shipped feature (ambiguous default weight in the Styles panel,
inconsistent casing), not data loss or a broken export.

## Findings
- **2026-07-12** — Root cause was evident from the report; fixed inline (no dispatch needed).
  - Added `weightNameFor(weight)` (`src/engine/type.mjs`, right after `siblingWeightDefaults`) — snaps
    an arbitrary weight to its nearest `WEIGHT_LADDER` stop and returns `{weight, name, slug}`.
  - `figma/binder/style-plan.mjs#stylePlans()`: when a voice/step has `sibs.length` (configured
    siblings), the core text-style name becomes `${voice}/${stepSlug}/${weightNameFor(s.weight).slug}`
    instead of the bare `${voice}/${stepSlug}`; sibling names changed from `${wv.name}` (title case) to
    `${wv.slug}` (the already-existing lowercase-kebab field). A voice/step with no siblings is
    unaffected — still bare.
  - Verified against the exact reported scenario: core weight 900 + siblings [800, 700] now emit
    `Display/xs/black`, `Display/xs/extra-bold`, `Display/xs/bold`.
  - Updated `test/engine/type.mjs` (`weightNameFor` coverage), `test/figma/style-plan.mjs` (renamed
    core/sibling assertions + a new bare-when-no-siblings case), `test/figma/plugin.mjs` (the e2e apply
    fixture's core/sibling names, plus a new assertion that the core's old suffixed name prunes and it
    reverts to bare once its siblings are dropped). `npm test` — 27/27 pass. `npm run build` — clean.
