# TKT-0010 — Control padding corrections: paddingWide/paddingNarrow (+compact variants) and the calibrated gap-unit

## Summary
Reporter-supplied correction to control-internal geometry (2026-07-16):

1. **Rename + reformulate the pads.** `padding` → **paddingNarrow** `= (height − icon)/2`
   (formula unchanged — the centering law); `edgePadding` → **paddingWide** `= (height − caret)/2`
   (NEW formula — was `height/2`; a bare-text control's edge now keys on the caret glyph, not the
   half-height, landing much tighter: md 14 → 7.5).
2. **Add compact variants:** **paddingWideCompact** `= (height − gap − caret)/2` and
   **paddingNarrowCompact** `= (height − gap − icon)/2`.
3. **Half-pixels stay EXACT** (ratified): the laws emit true halves (7.5 · 5.5 · 2.5 …) — valid
   CSS px and Figma floats; no rounding layer.
4. **gap is no longer `font/2`.** New hand-calibrated **GAP_UNIT** table (Desktop, canonical
   baseHeight 28): xs 3 · sm 3 · md 4 · lg 6 · xl 6 · 2xl 8, scaled by `baseHeight/28` × density;
   per-mode hand columns override (ratified matrix): Tablet frozen at Desktop; Mobile 3/3/4/5/5/6
   (reporter anchors xs 3 · 2xl 6); Desktop Lg 4/4/5/7/7/9; Desktop Xl 4/5/6/8/8/10; TV (BZZR)
   5/5/6/9/9/12.

Scope ruling: **engine + BZZR** (rename at the source, then migrate the live file — variables
rename in place, so node/style bindings survive by id).

## Acceptance
- `buildSize` returns `paddingNarrow`/`paddingWide`/`paddingNarrowCompact`/`paddingWideCompact`

  `opts.gapOverrides` (final per-size values, mirrors `fontOverrides`) winning.
